# API Documentation Audit — 2026-03-04

## Summary

Audited the OpenAPI spec generated by FastAPI across 83 endpoints (including /v1/ prefixed duplicates). All endpoints already had summaries and descriptions. The main gap was **missing error response codes** and **missing field descriptions** on Pydantic schemas.

## What Was Fixed

### 1. Error Response Codes Added to All Route Decorators

**Before:** Most endpoints only documented 200/201 + 422 (validation error).

**After:** All endpoints now document appropriate error responses:
- `401` — Missing or invalid authentication signature (on authenticated endpoints)
- `403` — Insufficient permissions (on owner-only endpoints)
- `404` — Resource not found (on single-resource endpoints)
- `409` — Conflict with current resource state (on job lifecycle endpoints)
- `429` — Rate limit exceeded (on all rate-limited endpoints)
- `503` — Withdrawals temporarily paused (on withdrawal endpoint)
- `400` — Invalid token (on auth verification endpoints)

Created `app/schemas/errors.py` with:
- `ErrorResponse` model (standard `{"detail": "..."}` body)
- Pre-built response dicts: `AUTH_ERRORS`, `OWNER_ERRORS`, `JOB_ERRORS`, `PUBLIC_ERRORS`, etc.

### 2. Field Descriptions Added to Pydantic Schemas

Added `description=` to fields in:
- `AgentCreate` — all fields now described
- `JobProposal` — seller_agent_id, listing_id, acceptance_criteria, requirements, max_budget, penalties, deadline, max_rounds
- `ListingCreate` — skill_id, description, base_price, currency, sla
- `ReviewCreate` — rating, tags, comment
- `WithdrawalCreateRequest` — amount, destination_address
- `DepositNotifyRequest` — tx_hash

### 3. Request Body Examples Added

- `AgentCreate` — added `json_schema_extra` with example registration payload

### 4. Lifespan Fix (Bonus)

Fixed `app/main.py` to respect the `deposit_watcher_enabled` config flag. Previously the deposit watcher was always started regardless of config, causing crashes in local dev environments without blockchain access.
